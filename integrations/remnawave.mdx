## Обзор

Бот управляет VPN подписками через API панели [Remnawave](https://github.com/remnawave/backend). Все операции с пользователями, серверами и трафиком проходят через единый API клиент.

## Настройка подключения

```env
REMNAWAVE_API_URL=https://panel.example.com/api
REMNAWAVE_API_TOKEN=your_panel_api_token
```

<Warning>
  Токен должен иметь полные права доступа к API панели.
</Warning>

## Возможности интеграции

| Функция | Описание |
| ------- | -------- |
| Создание пользователей | Автоматическое создание VPN-аккаунта при покупке подписки |
| Управление подписками | Продление, отключение, изменение параметров |
| Серверы и узлы | Просмотр статуса серверов, мониторинг нагрузки |
| Трафик | Отслеживание использования, лимиты, сброс |
| Устройства | Управление количеством подключённых устройств |
| Сквады | Группировка серверов для распределения пользователей |

## Архитектура

```
Telegram Bot  →  RemnaWaveAPI client  →  Remnawave Panel API
                 (app/external/          (HTTPS REST)
                  remnawave_api.py)
```

Клиент `RemnaWaveAPI` — асинхронный HTTP-клиент на базе `aiohttp`, поддерживающий:

- Автоматическую авторизацию через токен
- Retry при сетевых ошибках
- Парсинг ответов в типизированные dataclass-объекты

## Синхронизация

Бот поддерживает синхронизацию данных с панелью:

- **Серверы** — список доступных серверов и их статус
- **Пользователи** — сопоставление Telegram ID с аккаунтами панели
- **Подписки** — актуальные даты и параметры

Синхронизация запускается:
1. Автоматически при старте бота
2. По команде администратора через админ-панель
3. Периодически через фоновые задачи

## Мониторинг серверов

Бот периодически проверяет состояние серверов и отправляет уведомления администраторам при:

- Падении сервера (недоступен)
- Высокой нагрузке CPU/RAM
- Аномальном трафике
- Восстановлении после падения
