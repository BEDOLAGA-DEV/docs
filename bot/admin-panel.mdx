## Обзор

Админ-панель доступна в Telegram для пользователей, чьи Telegram ID указаны в `ADMIN_IDS`. Все разделы работают через инлайн-кнопки с пагинацией.

## Управление пользователями

### Поиск и фильтрация

| Фильтр | Описание |
| ------ | -------- |
| ID, username, имя | Текстовый поиск |
| Telegram ID | Прямой поиск по ID |
| По балансу | Выше/ниже порога |
| По тратам | Общая сумма покупок |
| По трафику | Использование трафика |
| По активности | Последняя активность |
| По кампании | Пользователи конкретной рекламной кампании |
| Готовы к продлению | Баланс >= `SUBSCRIPTION_RENEWAL_BALANCE_THRESHOLD_KOPEKS` |

### Действия с пользователем

| Действие | Описание |
| -------- | -------- |
| Изменить баланс | Начисление или списание |
| Изменить лимиты | Устройства, трафик, серверы |
| Сброс HWID | Сброс привязки устройств |
| Перегенерация подписки | Новые ключи подключения |
| Назначить промо-группу | Выбор из существующих |
| Назначить тариф | Привязка к тарифу |
| Купить подписку | Прямо из админки |
| Продлить/сократить | ±365 дней |
| Блокировка | С таймером, аудит действий |
| Ограничения | Блокировка функций (покупка, триал, пополнение) |
| Отправить сообщение | Прямое сообщение пользователю |
| Установить комиссию | Индивидуальный реферальный процент |

## Аналитика

### Дашборды

- Пользователи: всего, новые (сегодня/неделя/месяц), активные
- Подписки: активные, триальные, истёкшие, конверсии
- Платежи: по провайдерам, по суммам, по периодам
- Трафик: общий, по серверам, по пользователям

### Статистика продаж (Cabinet)

Расширенная аналитика доступна в **веб-панели Cabinet** — 6 вкладок с интерактивными графиками: сводка, триалы, продажи, продления, доп. услуги, пополнения. Подробнее: [Статистика продаж](/cabinet/sales-stats).

### Отчёты

```env
ADMIN_REPORTS_ENABLED=true
ADMIN_REPORTS_SEND_TIME=10:00
```

Ежедневные отчёты с ключевыми метриками в отдельный топик/чат.

## Remnawave

### Синхронизация

| Операция | Описание |
| -------- | -------- |
| Пользователи → панель | Синхронизация из бота в RemnaWave |
| Панель → бот | Синхронизация из RemnaWave в бота |
| Авто-синхронизация | По расписанию (`REMNAWAVE_AUTO_SYNC_TIMES`) |
| Серверы/Сквады | Синхронизация списка серверов |
| Миграция сквадов | Массовый перенос между серверами |

### Управление из бота

- Просмотр нод и сквадов
- Создание/редактирование сквадов
- Детальная статистика по нодам

## Уведомления в топики

Подробнее: [Топик-уведомления](/bot/notifications#топик-уведомления-для-администраторов).

| Топик | Переменная | Содержимое |
| ----- | ---------- | ---------- |
| Общий | `ADMIN_NOTIFICATIONS_TOPIC_ID` | Покупки, триалы, промокоды, версии |
| Тикеты | `ADMIN_NOTIFICATIONS_TICKET_TOPIC_ID` | Тикеты поддержки и SLA |
| NaloGO | `ADMIN_NOTIFICATIONS_NALOG_TOPIC_ID` | Чеки и налоги |
| Трафик | `SUSPICIOUS_NOTIFICATIONS_TOPIC_ID` | Подозрительная активность |
| Вывод | `REFERRAL_WITHDRAWAL_NOTIFICATIONS_TOPIC_ID` | Заявки на вывод |
| Бэкапы | `BACKUP_SEND_TOPIC_ID` | Файлы бэкапов |
| Логи | `LOG_ROTATION_TOPIC_ID` | Архивы логов |
| Отчёты | `ADMIN_REPORTS_TOPIC_ID` | Ежедневные отчёты |
