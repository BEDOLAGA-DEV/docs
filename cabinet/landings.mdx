## Обзор

Лендинги (Landing Pages) — публичные страницы покупки VPN-подписки, доступные по прямой ссылке без необходимости регистрации в Telegram. Покупатель выбирает тариф, период, способ оплаты, вводит email или Telegram-контакт и оплачивает подписку.

### Возможности

- Публичная страница по уникальному slug: `https://cabinet.example.com/quick/{slug}`
- Выбор из разрешённых тарифов и периодов
- Все подключённые платёжные системы (с возможностью ограничения)
- Покупка для себя или в подарок
- Мультиязычный интерфейс (заголовки, описания, баджи)
- Система скидок с таймером обратного отсчёта
- Кастомный CSS
- SEO meta-теги
- Автосоздание аккаунта в Cabinet с отправкой пароля на email

## Требования

<Warning>
  Для работы лендингов необходимо:
  1. `CABINET_ENABLED=true`
  2. `CABINET_URL` заполнен (используется для формирования ссылок)
  3. **SMTP настроен и работает** — для отправки данных подписки и пароля от Cabinet на email покупателя
  4. Хотя бы одна платёжная система подключена
  5. Настроены тарифы (`SALES_MODE=tariffs`)
</Warning>

### Почему SMTP обязателен

Когда пользователь покупает подписку через лендинг по email:
1. Создаётся аккаунт в Cabinet с автосгенерированным паролем
2. На email отправляется письмо с данными для входа и ссылкой подписки
3. Без SMTP пользователь не получит доступ к подписке

Настройка SMTP: [Настройка SMTP](/getting-started/smtp-setup)

## Создание лендинга

### Через Admin Cabinet

1. Откройте Cabinet → Админ-панель → **Лендинги**
2. Нажмите **Создать лендинг**
3. Заполните:

| Поле | Описание | Обязательно |
| ---- | -------- | ----------- |
| Slug | URL-путь лендинга (латиница, цифры, дефис) | Да |
| Заголовок | Мультиязычный заголовок страницы | Да |
| Подзаголовок | Мультиязычный подзаголовок | Нет |
| Тарифы | Выбор из доступных тарифов | Да |
| Периоды | Доступные периоды по тарифам (по умолчанию — все) | Нет |
| Способы оплаты | Разрешённые платёжные методы | Да |
| Подарки | Включить покупку в подарок | Нет |
| Особенности | Список фич с иконками (для промо-блока) | Нет |
| Footer | Текст внизу страницы | Нет |
| CSS | Кастомный CSS для стилизации | Нет |
| Meta | SEO title и description | Нет |

4. Нажмите **Сохранить**

Лендинг будет доступен по адресу: `https://cabinet.example.com/quick/{slug}`

### Ограничение периодов

По умолчанию на лендинге доступны все периоды тарифа. Можно ограничить через поле **Допустимые периоды**:

```json
{
  "15": [30, 90],
  "23": [30, 90, 180]
}
```

Где ключ — ID тарифа, значение — список доступных периодов (дни).

### Ограничение платёжных методов

Выберите только те платёжные системы, которые должны быть доступны на конкретном лендинге. У каждого метода можно настроить sub-options (например, отдельно Карта и СБП для PayPalych).

## Система скидок

Лендинги поддерживают временные скидки с таймером обратного отсчёта.

### Настройка скидки

В редакторе лендинга:

| Поле | Описание |
| ---- | -------- |
| **Процент скидки** | Глобальная скидка (1-99%) на все тарифы |
| **Начало скидки** | Дата и время начала действия (UTC) |
| **Окончание скидки** | Дата и время окончания (UTC) |
| **Текст баджа** | Мультиязычный текст на баннере скидки |
| **Переопределения по тарифам** | Индивидуальный процент скидки для каждого тарифа |

### Переопределения по тарифам

Если для тарифа задано индивидуальное значение скидки, оно используется вместо глобального:

```json
{
  "15": 30,
  "23": 50
}
```

Тариф 15 получит скидку 30%, тариф 23 — 50%, остальные — глобальный процент.

### Таймер обратного отсчёта

На публичной странице лендинга отображается баннер с:
- Текстом баджа (или процентом скидки по умолчанию)
- Таймером обратного отсчёта до окончания скидки (дни, часы, минуты, секунды)
- Перечёркнутой старой ценой и новой ценой со скидкой

Когда таймер истекает:
- Баннер скидки исчезает с анимацией
- Цены автоматически обновляются до полных
- Данные лендинга перезагружаются с сервера

### Расчёт скидки

Формула: `discounted_price = max(1, price - price * discount_percent // 100)`

- Целочисленное деление (копейки)
- Минимальная цена — 1 копейка
- На бэкенде и фронтенде используется одинаковая формула

## Процесс покупки

### Покупка по email

1. Покупатель открывает `https://cabinet.example.com/quick/{slug}`
2. Выбирает тариф, период, способ оплаты
3. Вводит email
4. Оплачивает
5. Бот:
   - Создаёт аккаунт в Cabinet (если не существует)
   - Генерирует пароль
   - Создаёт подписку в Remnawave
   - Отправляет email с данными для входа и ссылкой подписки
6. Покупатель получает email и может войти в Cabinet

### Покупка по Telegram

1. Покупатель вводит свой Telegram username
2. После оплаты бот отправляет сообщение в Telegram с данными подписки
3. Если у пользователя нет аккаунта бота — подписка ожидает активации

### Подарочная покупка

1. Покупатель выбирает «В подарок»
2. Вводит свои контакты и контакты получателя
3. После оплаты подписка активируется на аккаунте получателя

## Управление лендингами

### Список лендингов

Admin Cabinet → **Лендинги** — список всех лендингов с информацией:
- Slug и заголовок
- Статус (активен / неактивен)
- Количество покупок
- Дата создания

### Активация / деактивация

Кнопка toggle в списке лендингов. Неактивные лендинги возвращают 404 при попытке открыть.

### Удаление

Удаление доступно только для лендингов без покупок.

## Уведомления

При новых покупках через лендинг:
- Администраторы получают уведомление в Telegram (если `ADMIN_NOTIFICATIONS_ENABLED=true`)
- Покупатель получает email/Telegram сообщение с данными подписки

## Пример .env

```env
# === ОБЯЗАТЕЛЬНЫЕ ДЛЯ ЛЕНДИНГОВ ===
CABINET_ENABLED=true
CABINET_URL=https://cabinet.example.com
SALES_MODE=tariffs

# === SMTP (обязательно для email-покупок) ===
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_USER=noreply@example.com
SMTP_PASSWORD=your_password
SMTP_FROM_EMAIL=noreply@example.com
SMTP_FROM_NAME=My VPN Service

# === Платёжная система (хотя бы одна) ===
YOOKASSA_ENABLED=true
YOOKASSA_SHOP_ID=123456
YOOKASSA_SECRET_KEY=your_secret
```

## Устранение проблем

### Лендинг возвращает 404

1. Лендинг активен? Проверьте статус в Admin Cabinet → Лендинги
2. Slug указан верно в URL?
3. `CABINET_ENABLED=true`?

### Покупатель не получает email

1. SMTP настроен? Проверьте [Настройка SMTP](/getting-started/smtp-setup)
2. Письмо в спаме?
3. Проверьте логи бота на ошибки отправки:
   ```bash
   docker logs <bot_container> 2>&1 | grep -i smtp
   ```

### Скидка не отображается

1. Текущее время между `discount_starts_at` и `discount_ends_at`?
2. `discount_percent` задан (1-99)?
3. Даты указаны в UTC?

### Подписка не создаётся после оплаты

1. Remnawave панель доступна? Проверьте `REMNAWAVE_API_URL`
2. Тариф существует в панели?
3. Проверьте логи:
   ```bash
   docker logs <bot_container> 2>&1 | grep -i "guest_purchase"
   ```
